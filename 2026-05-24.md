## General Ideas (5)

**Idea 1: MedCabinet**
Every household has a medicine cabinet full of expired ibuprofen, three half-used tubes of the same cream, and no idea what treats what — people only look inside when they're sick and desperate. MedCabinet is a shared family medicine inventory: scan or log medications, track expiry dates, note what each item is used for, and get a nudge when something runs out or expires. A dead-simple shared household tool that saves the "do we have anything for a headache?" panic at midnight.

**Idea 2: TenantLog**
Renters dealing with slow or unresponsive landlords have no organized record of maintenance requests, promised timelines, and actual response dates — when a dispute reaches a tenancy tribunal, all they have is a chaotic WhatsApp thread. TenantLog is a tenant-side maintenance diary: log every request with date and photo, track landlord responses, generate a clean printable timeline of the issue history. Not a platform or marketplace — just the documentation layer that protects renters and gives them leverage in disputes.

**Idea 3: OutdoorBuddy**
Solo hikers and outdoor enthusiasts want to find trail partners for the weekend, but Meetup is too formal, local Facebook groups are chaos, and fitness apps don't do casual coordination. OutdoorBuddy matches you with nearby hikers by trail difficulty preference, fitness level, and availability — think "Bumble BFF for trails." A simple radius-based match and lightweight group chat for planning a hike this Saturday, no ongoing social feed required.

**Idea 4: RecurringLocal**
The best local regular events — Wednesday pub quiz, Sunday organic market, monthly sketch night at the café — are almost impossible to discover because they're not ticketed, not on Eventbrite, and posted on a Facebook event nobody sees. RecurringLocal is a crowd-sourced directory of weekly and monthly local recurring events: add what you know, follow your area, and get a Sunday digest of what's happening your city this week. The antiTicketmaster for the stuff that's just... always there.

**Idea 5: GradFund**
When a grandchild, niece, or nephew graduates, the family wants to pool a meaningful gift — but coordinating across aunts, uncles, and grandparents by WhatsApp is a logistical nightmare of chased payments and duplicate contributions. GradFund is a one-time group gift collection tool for family milestones: create a fund, set a target, share a link, track who's contributed, and when the goal is hit it suggests or facilitates the gift. No subscription — charge a small facilitation fee or optional premium for physical card delivery.

## Developer Tools (3)

**Idea 6: APIChangeAlert 🛠️ DEV TOOL**
Developers building on third-party APIs (Stripe, OpenAI, Twilio, etc.) regularly find out about breaking changes, deprecations, or rate limit updates too late — long after the changelog was quietly published or the GitHub issue was closed. APIChangeAlert monitors your specified external API providers' changelogs, GitHub release notes, and status pages, maps detected changes to the APIs you've declared as dependencies, and sends you a Slack/email alert before breaking changes hit production. The "no more surprised by a v2 migration" service that takes 5 minutes to set up.

**Idea 7: FeatureFlagAudit 🛠️ DEV TOOL**
Codebases accumulate feature flag debt — flags that were added for a launch experiment, never cleaned up, and now nobody knows if they're still active, always-on, or dead code. FeatureFlagAudit scans your repo for flag definitions and usages, generates a cleanup report (stale flags, always-true flags, unreferenced flags), and provides a lightweight hosted dashboard to see the full flag inventory across environments. Works vendor-agnostic — reads your existing LaunchDarkly, Unleash, or homegrown boolean configs. The "take back control of your toggle graveyard" tool.

**Idea 8: MCPPublish 🛠️ DEV TOOL**
With Model Context Protocol (MCP) servers exploding in 2026, developers building MCP tools have no standardized way to publish, version, and distribute them — most live as undiscoverable GitHub repos with manual install instructions. MCPPublish is a registry and publish workflow for MCP servers: one CLI command to publish, automatic versioning, hosted documentation page, and a searchable directory users can browse to find MCP tools by capability. The npm for MCP — makes your server discoverable, installable, and maintainable in the ecosystem everyone is rushing to build.
